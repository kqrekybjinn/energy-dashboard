## 4. 现有数据库 Schema

`supabase/schema.sql` 当前只定义了 `profiles` 表：
- `id` uuid PK
- `user_id` uuid FK → auth.users (unique)
- `username` text unique
- `display_name` text
- `avatar_url` text
- `bio` text
- RLS 已启用：所有人可读，只能管理自己的 profile

如果需要新增表（如能耗记录、设备表等），请先在 `schema.sql` 中写好 DDL，然后通过 Supabase Dashboard SQL Editor 执行，或本地 `supabase db push`。

## 7. 发布流程

1. 修改代码
2. 语法检查：`node --check app.js`
3. 更新 `version.json` 版本号
4. 提交到 `main`
5. `git push origin main`
6. GitHub 自动触发 Pages build
7. 确认线上 `version.json` 可访问

---

## 8. 注意事项

- 不要提交 service_role key、GitHub OAuth secret、Supabase access token
- 前端功能通常走 Supabase HTTPS REST API，不需要直连 Postgres（5432 端口可能超时）
- `.github/workflows/pages.yml` 不要随便创建——当前是 legacy Pages 模式
- 本机访问 GitHub 可能需要代理：`export HTTP_PROXY='http://127.0.0.1:7897' HTTPS_PROXY='http://127.0.0.1:7897' ALL_PROXY='socks5://127.0.0.1:7897'`
